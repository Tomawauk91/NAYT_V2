[Intégration] : Lister les commandes CLI prioritaires et leurs arguments

Les CLI prioritaires automatisables :

1- Reconnaissance réseau :

Nmap (priorité absolue) : Découverte hôtes, ports, services, versions

Commandes :

nmap -sS -p- -T4 <target>
nmap -sV -O <target>
nmap --script vuln <target>

Arguments :

* -sS : scan SYN rapide
* -p- : tous les ports
* -sV : détection des versions
* -O : OS fingerprint
* --script vuln : scripts vuln NSE
* -oX / -oA : export XML pour parsing


2- Reconnaissance web :

WhatWeb : Fingerprinting technologique

Commandes :

whatweb <url>
whatweb -a 3 <url>

Argument :

* -a : agressivité


Nikto : Vulnérabilités web connues

Commandes :

nikto -h <url>
nikto -h <url> -Tuning x 6

Arguments :
* -h : cible
* -Tuning : réduire bruit / faux positifs
* -Format json (si dispo)


3- Enumération web & contenu :

Gobuster / Feroxbuster : Bruteforce de chemins

Commande :

gobuster dir -u <url> -w <wordlist>

Arguments :

* dir / dns
* -w : wordlist
* -x : extensions
* -t : threads
* -o : output


4- Failles applicatives (semiautomatisé) :

Sqlmap : Détection/exploitation SQLi

Commande :

sqlmap -u <url> --batch

Arguments :

* -u : cible
* --batch : non interactif (important pour Celery)
* --risk / --level
* --dump


5- Authentification & mots de passe :

John the Ripper : Cassage de hash offline

Commandes :

john hashes.txt
john --wordlist=rockyou.txt hashes.txt

Arguments :

* --wordlist
* --rules
* --format


6- Postexploitation :

Hydra : Bruteforce en ligne

Commande :

hydra -l user -P pass.txt ssh://<target>

Arguments :

* protocole
* login / password
* threads



Les CLI secondaires automatisables :

1- curl : Validation HTTP / API

Rôles :

* Vérifier la présence d’un service web
* Tester headers, méthodes HTTP
* Valider une vulnérabilité détectée

Commandes : 

curl -I https://target
curl -X GET https://target/api
curl -X POST -d "param=value" https://target/login

Arguments :

* -I : headers uniquement
* -X : méthode HTTP
* -d : données POST
* -H : headers custom
* -k : ignorer TLS
* -L : suivre redirections
* --max-time : timeout


2- netcat (nc) : Vérification de services

Rôles :

* Tester si un port répond réellement
* Banner grabbing simple
* Test TCP/UDP ciblé

Commandes :

nc -vz target 22
nc target 80

Arguments :

* -v : verbose
* -z : scan sans envoi de données
* -u : UDP
* -w : timeout


3- sslscan / testssl.sh : TLS complémentaire

Rôles :

* Vérifier versions TLS
* Cipher suites faibles

Commandes :

sslscan target:443
testssl.sh https://target

Arguments :

* --no-colour
* --jsonfile
* --severity


4- Whois : Informations domaine/IP

Rôles :
* Infos registrars, plages IP
* OSINT rapide

Commande :

whois example.com

Arguments :

* -H : pas d’entêtes légales


5- dig / nslookup : DNS

Rôles :

* Résolution DNS
* Enumération basique

Commandes :

dig example.com
dig MX example.com

Arguments :

* +short
* ANY
* MX, TXT, NS


6- traceroute / tracepath : Chemins réseau

Rôles : 

* Identifier la topologie réseau
* Détection filtrage

Commande :

traceroute target

Arguments :

* -n : pas de DNS
* -w : timeout

7- enum4linux (si AD / SMB) :

Rôle :

* Enumération SMB / AD basique

Commande :

enum4linux -a target

Arguments :

* -a : tout
* -u / -p : creds


8- smbclient : SMB ciblé

Rôles :

* Lister partages
* Vérifier accès anonyme

Commande :

smbclient -L //target -N

Arguments : 

* -L : list
* -N : no password


9- ftp / ssh : Vérification d’accès

Rôles :

* Tester accès anonymes
* Vérifier bannières

Commandes :

ftp target
ssh user@target

Arguments clés :

* -v
* -o StrictHostKeyChecking=no
